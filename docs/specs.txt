Architected phased SaaS roadmap, flagged browser limitations, prioritized implementation sequence
This is a full platform, not a single build — let's sequence it right. Two things first:

Reality check on #19: Browsers can't fully block OS-level keyboard shortcuts (Alt+Tab, Cmd+Tab, Win key) or remove the OS window chrome — that's outside any web app's sandbox, even native apps struggle with this on locked-down student devices. What we can do realistically:

Fullscreen API (forces fullscreen, exits are detected and logged/flagged)
Detect tab-switch / window-blur / visibility-change → auto-pause test + flag attempt + warn user
Disable right-click, text-selection, copy-paste within the exam screen
Block browser back button via history trapping
Kick user out and lock the attempt if fullscreen is exited 2-3 times
This gets you 90% of the deterrent effect real exam centers rely on, without promising something the browser physically can't do. Worth setting that expectation now rather than after you build.

Build sequence (mapping your 19 objectives):

Phase	Covers	Why this order
0 – Foundation	DB schema, auth (Google + mobile OTP)	Everything depends on this
1 – Core flow	Profile CRUD, dashboard, instructions/info/disclaimer pages, free test unlock logic	Gets a user end-to-end on the free test — your validation loop
2 – Exam Engine	Real exam UI, countdown timer, fullscreen + anti-cheat, auto-submit	The actual product differentiator, needs to be rock solid before anything else
3 – Analysis	Post-exam report, weak areas, comparison vs other free-test takers, purchase/practice history	Makes the free test convert to paid
4 – Ecom	Packages, cart, billing, payment, email notifications, dashboard gating by package	Revenue layer
5 – Admin	User view/update, Excel exam upload, AI-generated exams, revenue/analytics dashboard	Ops layer, needed before scale not before first user
6 – Hardening	Rate limiting, session security, anti-scraping on question bank, audit logs	Ongoing, but do a pass before real money moves
Given the size (this is 8-10 weeks of solo dev, not a one-shot artifact), I'd build this properly in a repo via Claude Code rather than chat files — same as your TOP setup (Fastify/Prisma/Postgres backend + web frontend).
